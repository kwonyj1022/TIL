멀티미디어 설정하기 - audio 태그, video 태그

audio 태그 자세한 설명

  • 항상 src 속성과 함께 사용하며, src 속성값은 삽입하려고 하는 오디오 파일의 경로
  • 오디오 컨트롤 패널이 웹브라우저에 노출되도록 controls 속성을 설정해야 함

video 태그 자세한 설명

  • video 태그도 비디오 경로를 입력하는 src 속성을 필수로 사용해야 함
  • controls 속성을 추가해 사용자가 제어할 수 있는 컨트롤 패널도 표시해야 함

웹 브라우저 MP4 WebM Ogg

파일 포맷 미디어 타입

source 태그 : audio 태그와 video 태그에서 리소스(파일)의 경로와 미디어 타입을 명시하는 데 사용 자세한 설명

  • 멀티미디어 태그를 사용할 때 필수는 아니지만, 되도록 함께 사용하면 좋음
    (웹 접근성을 고려해 source 태그로 다양한 포맷의 멀티미디어를 등록해 두는 방식으로 사용하는 것이 좋음)
  • 이유 : 예를 들어 OGG 포맷을 우선지원하고 어쩔 수 없는 경우에만 MP3 포맷을 지원해야 하는 경우,
    source 태그를 사용하면 먼저 작성한 source 태그의 파일부터 현재 웹 브라우저에서 해당 파일 포맷을 지원하는지 확인하고,
    만약 웹 브라우저에서 지원하지 않는다면 그 다음에 작성된 source 태그 파일을 같은 방식으로 확인하고,
    모든 source 태그를 확인했는데, 전부 지원하지 않는 형식이라면 마지막에 작성한 텍스트가 사용자에게 노출됨
